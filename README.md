# IMG Squeeze CLI (imsq)

カレントディレクトリ内の画像を最適化（圧縮・リサイズ・フォーマット変換）し、指定フォルダへ一括出力するCLIツールです。
このツールはAIを使用して作成しました。各環境での動作保証はありません。

## 特徴

- **簡単な一括最適化** — カレントディレクトリ内の画像をスキャンし、まとめて処理します
- **多様なフォーマット対応** — `jpg`, `png`, `gif`, `webp` への変換をサポート
- **ファイルサイズ制限** — 目標サイズを指定すると品質を自動調整して収まるように圧縮
- **柔軟なリサイズ** — ピクセル指定・パーセント指定・ストレッチに対応
- **メタデータ保持** — Exif・ICCプロファイルなどを残すオプション
- **連番リネーム** — 桁数を `*` の数で指定できるパターン形式
- **対話モード** — 処理対象の画像をインタラクティブに選択
- **出力先指定** — 任意のディレクトリへ出力可能
- **再帰処理** — サブディレクトリ内の画像もディレクトリ構造を維持して処理
- **安全設計** — 既定では `optimized/` などの出力先フォルダへ保存

## インストール

```bash
git clone https://github.com/naorbieta/imgsqueeze-cli.git
cd imgsqueeze-cli
npm install
npm run build
npm link
```

`npm link` で任意のディレクトリから `imsq` コマンドが使えるようになります。

## 使い方

```bash
imsq [オプション]
```

### 直前の設定を引き継ぐ

引数なしで `imsq` を実行すると、前回使ったオプションを再利用します。
ただし `-p` と `-c` は引き継がれず、使うたびに毎回明示指定が必要です。

### オプションの初期化

`init` は、前回設定を引き継いだ状態をその場で解除したいときに使います。
通常の CLI 引数を1つでも指定した実行では前回設定は引き継がれないため、`imsq -d init` のような書き方ではほとんど意味がありません。

```bash
imsq -c
# 確認プロンプト内で
init
# または個別に解除
-d init -l init
```

### リネームのみ

`-n` だけを指定した場合は、圧縮せずに元ファイルをコピーしてリネームします。

### オプション一覧

| オプション | 短縮形 | 説明 |
| :--- | :--- | :--- |
| `--format <type>` | `-f` | 出力フォーマット変換 (`jpg`, `png`, `gif`, `webp`) |
| `--size <size>` | `-s` | 目標ファイルサイズ上限 (例: `100kb`, `1mb`) |
| `--length <dims>` | `-l` | リサイズ指定（後述） |
| `--recursive` | `-r` | サブディレクトリ内の画像も処理する |
| `--keep` | `-k` | メタデータを保持する (Exif, ICC profileなど) |
| `--name <pattern>` | `-n` | リネームパターン (`?` = 元ファイル名, `*` の数で連番桁数を指定) |
| `--pick` | `-p` | 対話モード: 処理対象の画像を選択する |
| `--directory <dir>` | `-d` | 出力先ディレクトリ指定 |
| `--confirm` | `-c` | 処理を開始する前に確認を挟む (`-p` と併用時は選択後) |
| `--hard` |  | 出力成功後に元ファイルをゴミ箱へ送らず削除する |
| `--trash` |  | 出力成功後に元ファイルをゴミ箱へ送る |
| `--version` | `-v` | バージョン表示 |
| `--help` | `-h` | ヘルプ表示 |

## 使用例

### フォーマット変換

```bash
# すべての画像を WebP に変換
imsq -f webp
```

### ファイルサイズ制限

```bash
# 100KB 以下になるよう品質を自動調整して圧縮
imsq -s 100kb
```

品質を段階的に下げて目標サイズに収まるよう試行します。最低品質でも超過する場合は警告が表示されます。

### リサイズ

```bash
# 横幅を最大 600px にリサイズ（アスペクト比維持）
imsq -l w:600

# 高さを最大 400px にリサイズ（アスペクト比維持）
imsq -l h:400

# 横幅 50% に縮小（元画像に対するパーセント指定）
imsq -l w:50%

# 横幅と高さを両方パーセント指定（アスペクト比維持）
imsq -l w:50%,h:50%

# 横幅・高さを両方ピクセル指定 → ストレッチ（アスペクト比無視）
imsq -l w:200,h:100
```

> **ストレッチについて**
> `w` と `h` の両方をピクセル数で指定した場合のみ、アスペクト比を無視してその寸法ぴったりに変形します。
> パーセント指定や片方のみ指定の場合はアスペクト比を維持します。

### メタデータ保持

```bash
# Exif や ICC プロファイルを保持したまま圧縮
imsq -k
```

### 連番リネーム

`?` で元ファイル名、`*` の数で連番の桁数を指定します。

```bash
# example.jpg -> example_optimize.jpg
imsq -n "?_optimize"

# img_001.jpg, img_002.jpg, ... と出力
imsq -n "img_***"

# example.jpg -> example_01.jpg, sample.jpg -> sample_02.jpg
imsq -n "?_**"

# photo_01.png, photo_02.png, ... と出力
imsq -n "photo_**" -f png

# 1.jpg, 2.jpg, 10.jpg, ... と出力（桁あわせなし）
imsq -n "*"
```

### 対話モード

```bash
imsq -p
```

起動するとスキャン後に一覧が表示され、スペースキーで処理したい画像を選択、Enterで確定できます。  
EscキーまたはCtrl+Cでキャンセルします。

### 確認モード

```bash
imsq -c
```

処理を開始する前に確認を挟みます。`-p` と併用した場合は、先に画像を選択してから確認画面に進み、選択されたファイル名も表示されます。`y` / `n` に加えて、ここで追加のオプションを入力してその場で上書きできます。他のオプションと組み合わせて使えます。
`init` を単体で入力すると、その実行中の設定をいったん初期化し、`-d init` のように個別初期化もできます。

```bash
# 対話で選択してから確認を経て処理開始
imsq -p -c -f webp
```

### 出力先ディレクトリ指定

```bash
# カレントディレクトリ内の ./output フォルダへ出力
imsq -d ./output

# カレントディレクトリそのものへ出力（上書きに注意）
imsq -d .

# 絶対パス指定
imsq -d /Users/me/Pictures/compressed
```

未指定の場合、カレントディレクトリ内に `optimized/`（重複時は `optimized_1/` など）が自動生成されます。

`-d .` などで出力先が元ファイルと同一パスになる場合、既定では元ファイルをゴミ箱へ送ってから差し替えます。

### 元ファイルの処理

```bash
# 出力成功後、元ファイルをゴミ箱へ送る
imsq -d . -f webp --trash

# 出力成功後、元ファイルをゴミ箱へ送らず削除する
imsq -d . -f webp --hard
```

- `--trash` は、出力先が別ファイルでも出力成功後に元ファイルをゴミ箱へ送ります
- `--hard` は、出力先が別ファイルでも出力成功後に元ファイルを削除します
- `--hard` と `--trash` は同時に指定できません

### 組み合わせ例

```bash
# サブディレクトリも含めて再帰処理、WebP に変換、横幅 1280px にリサイズ、メタデータ保持
imsq -r -f webp -l w:1280 -k

# 対話で画像を選択し、100kb 以下に圧縮して ./dist へ出力
imsq -p -s 100kb -d ./dist

# 画像を選択して連番リネームし thumb_001.jpg 形式で出力
imsq -p -n "thumb_***" -f jpg
```
